## 1. 사이트 개요

### 컨셉
"다양한 세계관을 수집하는 아카이브"  
여러 세계관의 캐릭터·아트·에셋을 한 곳에 모아둔 크리에이터의 작업 기록소.

### 핵심 목표
- 이 사이트 하나로 역량 증명 + 나를 원하게 만들기 + 연락 유도
- 이력서·포트폴리오·브랜딩을 하나의 URL로 대체

## 4. 섹션별 상세 기획

---

### 4-1. 랜딩 (Hero Section)

**목적:** 첫인상 + 정체성 전달

**구성 요소**
- 풀스크린 다크 배경
- 대표 캐릭터 일러스트 or 파티클 연출
- 이름 + 포지션 한 줄
  - 예: `AI Art Director / Content Creator / Vibe Coder`
- 브랜드 카피라이트 한 줄
  - 예: *"기획하고, 그리고, 만든다 — AI와 함께"*
- 스크롤 유도 화살표

---

### 4-2. About

**목적:** 나는 누구인가 — 브랜드 인물 소개

**구성 요소**
- 프로필 사진 or 자화상 일러스트
- 3~5줄 자기소개 (자소서 말투 아닌 브랜드 카피 문체)
- 기본 인적사항 (나이 / 전공 / 현재 상태 간략히)
- `Open to Work` 상태 뱃지 (구직 의사 명시)

---

### 4-3. Skills

**목적:** 역량 시각화

**구성 요소**
- 스탯바 or 태그 형식으로 핵심 역량 표시
- 표시할 항목 예시:
  - AI 아트 디렉팅
  - 캐릭터 기획·설정
  - 스토리텔링
  - 바이브 코딩
  - 콘텐츠 기획
  - 세계관 설계

> 💡 RPG 스탯창 UI로 구성하거나 미니멀 바 형식 중 선택

---

### 4-4. Works ← 메인 섹션

**목적:** AI 아트 엔지니어링 역량 + 다양한 작업물 전시

#### 탭 구조
```
캐릭터  |  배경·환경  |  에셋  |  3D  |  기타
```

---

#### [캐릭터 탭] — 세력별 갤러리

- 세력 선택 탭 상단 배치 (심해 / 마법학원 / 판타지 / ...)
- 탭 선택 시 포인트 컬러 전환 + 해당 세계관 캐릭터 카드 나열
- 각 세계관당 캐릭터 5장 목표

**카드 클릭 시 프로필 팝업**
| 항목 | 내용 |
|------|------|
| 캐릭터명 | — |
| 세계관 소개 | 1~2줄 |
| idle 영상 | 캡컷으로 제작, 5초 내외, WebM 포맷 권장 |
| 제작 툴 | 사용한 AI 툴 명시 |
| 제작 노트 | 이 아트풍을 선택한 의도 한 줄 |

**세계관별 아트 스타일**
| 세계관 | 아트풍 |
|--------|--------|
| 심해 | 일본풍 |
| 마법학원 | 미국 카툰풍 |
| 판타지 모험 | 픽셀풍 |
| *(추가)* | 별도 지정 |

> ⚠️ 각 세계관 내 5장은 톤·스타일 일관성 유지 필수  
> (다양한 풍 = AI 디렉팅 폭, 내부 일관성 = AI 프롬프트 컨트롤 능력을 동시에 증명)

---

#### [배경·환경 탭]

- 풀샷 이미지 그리드 나열
- 클릭 시 확대 + 세계관 태그 표시
- 어느 세계관의 배경인지 캐릭터 탭과 연결

---

#### [에셋 탭]

- 아이콘 / UI킷 / 오브젝트 등 묶음 카드 형식
- 배경 투명 PNG 썸네일로 표시
- "실제 게임·콘텐츠에 활용 가능한 형태" 강조

---

#### [3D 탭]

- 베스트 앵글 렌더 이미지 2~3장 카드 표시
- 턴테이블 GIF 삽입 (Blender 렌더, 용량 경량화)
- 카드 하단 **Sketchfab 링크 버튼** → 외부 3D 뷰어로 이동
  - 뷰어 무게를 사이트 밖으로 위임하는 구조

---

#### [기타 탭]

- 분류 어려운 실험적 작업물
- 영상 클립: YouTube / Vimeo 임베드
- 생성 배경·효과음·기타 AI 실험물 등

---

### 4-5. Projects (바이브코딩)

**목적:** 기획력 + 실행력 증명

**구성 요소**
- 바이브 코딩으로 만든 프로젝트 카드형 나열
- 각 카드 구성:
  - 스크린샷 or 미리보기
  - 프로젝트명 + 한 줄 설명
  - "기획 의도 → 결과물" 흐름 짧게
  - GitHub / 데모 링크 버튼

---

### 4-6. Achievements (경력·수료·연재)

**목적:** 공신력 있는 외부 검증 제시

**구성 요소 (트로피·배지·타임라인 형식)**

| 항목 | 내용 |
|------|------|
| 소설 연재 | 네이버 / 카카오 정식 연재, 플랫폼 링크 포함 |
| 한국콘텐츠진흥원 | 뉴콘텐츠아카데미 2기 우수 수료 |
| 대기업 협업 | 협업 내용 (공개 가능한 선에서) |
| 학력 | 게임콘텐츠과 재학 |

> 💡 RPG 업적 해금 UI로 연출하면 포트폴리오 전체 컨셉과 자연스럽게 연결됨

---

### 4-7. Contact

**목적:** 채용·협업·의뢰 유입 창구

**구성 요소**
- `"협업·의뢰·채용 문의 환영합니다"` 문구 명시
- 이메일 주소 (클릭 시 메일앱 열리게)
- GitHub 링크
- 인스타 or 작업물 SNS 채널
- **문의 폼** (이름 / 이메일 / 메시지 / 전송 버튼)
  - 폼이 있어야 회사 홈페이지 느낌 완성됨
- **PDF 이력서 다운로드 버튼**

## 7. 제작 우선순위

```
1단계: 사이트 기본 구조 + 테마 완성
2단계: 캐릭터 작업물 세계관 1개 완성 (5장 + idle)
3단계: Works 섹션 완성 (전 탭)
4단계: About / Skills / Achievements 작성
5단계: Contact 폼 연동
6단계: 추가 세계관 확장
```

> ⚠️ 기능 욕심보다 작업물 퀄리티 우선  
> 캐릭터 카드 10개가 퀄리티 있게 채워진 뒤 오픈하는 것이  
> 기능 잔뜩 넣은 카드 3개짜리보다 훨씬 강력함
